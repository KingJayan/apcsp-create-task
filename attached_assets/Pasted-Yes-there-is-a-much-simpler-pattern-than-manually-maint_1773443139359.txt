Yes — there **is a much simpler pattern** than manually maintaining undo logic everywhere you mutate `waypoints`. The trick is to treat history as **state snapshots**, not operations.

Right now you mutate `waypoints` in dozens of places. Instead of tracking every mutation, you:

> **Snapshot the entire `waypoints` array before any change.**

Then undo/redo just restores snapshots.

Since waypoint arrays are small, copying them is cheap.

---

# Simple Undo/Redo Architecture

Use **two stacks**:

```js
let undoStack = [];
let redoStack = [];
```

### Snapshot helper

```js
function snapshot() {
    undoStack.push(structuredClone(waypoints));
    redoStack.length = 0; //clear redo on new change
}
```

Call `snapshot()` **before any mutation**.

---

# Undo

```js
function undo() {
    if (undoStack.length === 0) return;

    redoStack.push(structuredClone(waypoints));
    waypoints = undoStack.pop();

    updatePath();
    renderSidebarBlocks();
}
```

---

# Redo

```js
function redo() {
    if (redoStack.length === 0) return;

    undoStack.push(structuredClone(waypoints));
    waypoints = redoStack.pop();

    updatePath();
    renderSidebarBlocks();
}
```

---

# Keyboard handler

Replace your current undo code with:

```js
if (e.ctrlKey && e.key === "z") {
    e.preventDefault();
    undo();
}

if (e.ctrlKey && (e.key === "y" || (e.shiftKey && e.key === "Z"))) {
    e.preventDefault();
    redo();
}
```

---

# Where to call `snapshot()`

Only when a **user action begins**, not every frame.

Examples:

### Adding waypoint

```js
canvas.addEventListener("mouseup", (e) => {
    if (!isEditMode) {
        snapshot();
        waypoints.push({...});
    }
});
```

---

### Deleting waypoint

```js
btn.addEventListener('click', (e) => {
    snapshot();
    waypoints.splice(idx, 1);
});
```

---

### Dragging waypoint (important)

Take snapshot **on drag start**, not during move:

```js
canvas.addEventListener("mousedown", (e) => {
    if (currState === "running" || !isEditMode) return;

    snapshot(); //save state before drag begins
});
```

---

### Sidebar edits

```js
input.addEventListener('input', () => {
    snapshot();
    waypoints[idx].x = ...
});
```

*(or use `change` event instead so it fires once)*

---

# Why this approach is better

You avoid:

❌ manually reverting operations
❌ remembering what changed
❌ deep mutation tracking
❌ one-layer cache hacks

Instead you get:

✔ unlimited undo
✔ unlimited redo
✔ trivial logic
✔ works with **any future feature**

---

# Even Cleaner (Optional)

You can centralize **all mutations**:

```js
function modifyWaypoints(fn) {
    snapshot();
    fn(waypoints);
    updatePath();
    renderSidebarBlocks();
}
```

Then everywhere:

```js
modifyWaypoints(wp => {
    wp.push(newPoint);
});
```

or

```js
modifyWaypoints(wp => {
    wp.splice(idx, 1);
});
```

This guarantees history **never breaks**.

---

# Memory Cost (tiny)

Even **200 history states** of ~50 waypoints:

```
50 * ~40 bytes * 200 ≈ 400 KB
```

Nothing for modern browsers.

---

# One More Pro Tip (huge UX improvement)

Limit history size:

```js
const HISTORY_LIMIT = 200;

function snapshot() {
    undoStack.push(structuredClone(waypoints));
    if (undoStack.length > HISTORY_LIMIT) undoStack.shift();
    redoStack.length = 0;
}
```

✅ **Summary**

Best approach for your project:

* Snapshot whole `waypoints`
* Two stacks (`undo`, `redo`)
* Call snapshot on **user actions only**
* Restore arrays on undo/redo

This avoids rewriting all your logic.
